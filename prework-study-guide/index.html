HTML

    HTML creates the structure of a webpage. Its elements work together to provide content and the basic framework for all web pages, making it easy for users to navigate the webpage.
  • An element is a type of content on a webpage; they also divide content on the page and describe it to browsers and developers.
  • Some HTML elements are visible to users, and others provide important information about an HTML document and website used by the browser so that the page renders correctly.
  • This [!DOCTYPE html] is a declaration, it tells the browser what document type to expect when opened.
  • The [html] opening tag represents the root, or start, of our HTML document
  • The [head] element contains information about the webpage or metadata.
  • Metadata is webpage data the browser uses to effectively serve our webpage to users. Typically not seen by users but contains essential information about accessibility, search engines, and performance.
  • Let's examine the metadata elements in the [head] of our HTML code.
  • The [title] element defines the title of a webpage and is extremely important for Search Engine Optimization (SEO)
  • The [style] element defines the CSS styles associated with a website.
  • The [base] element defines the base URL for a webpage.
  • The [meta] element defines metadata such as the character set, description, keywords, author, and viewport.
  • The s[cript] element embeds client-side scripts in the HTML document.
  • The [link] element connects an external resource to the HTML document.
  • The [body] element defines the document's body, containing visible user content. All the content we want to display to users must be inside the body element.
  • Elements that are nested inside other elements are called child elements. The parent element that holds all the children, in this case, the [body], is the parent element.
  • Semantic HTML element clearly describes their meaning to both the browser and the developer.
  • It plays a vital role in ensuring your webpage is accessible to the widest variety of users, including those using assistive technologies such as screen readers.
  • Let's examine the semantic HTML elements in the [body] of our HTML code.
  • The [header] appears at the top of the page and usually contains a navigation menu and introductory content.
  • The [main] element Specifies the main content of a document.
  • The [nav] element defines a set of navigation links.
  • The [aside] element defines some content aside from where it is placed (like a sidebar).
  • The [section] element defines a section in a document.
  • The [article] element specifies independent, self-contained content.
  • The [footer] element defines a footer for a document or section.
